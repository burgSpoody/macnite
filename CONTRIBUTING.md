# Contributing to MacNite

Thanks for your interest in contributing to MacNite!

## How to contribute to MacNite

### How to contribute using issue templates

1. [Click here](https://github.com/burgspoody/macnite/issues/new/choose) or click "New issue" at the [issues](https://github.com/burgspoody/macnite/issues) tab in this repository
2. Select the type of issue that you want to make a contribution in
3. Fill in the fields provided (without erasing the fields provided)
4. Click on "Create"
- (Don't use bug report issue types while contributing!)

## Contributing using Pull Requests

- An email should be sent to macnite.email@icloud.com stating basic information about the pull request(s).
- The pull request should have major changes to be considered; minor changes can be suggested via [issues](https://github.com/burgspoody/macnite/issues/new/choose) only on suggestion issue templates.

## How to report bugs

1. [Click here to create an issue](https://github.com/burgspoody/macnite/issues/new/choose) or click "New issue" at the [issues](https://github.com/burgspoody/macnite/issues) tab in this repository
2. Select the type of issue template that you want to report a bug in
3. Fill in the fields provided (without erasing the fields provided)
4. Click on "Create"
- (Don't use contributing issue types while reporting bugs!)

### For Security Vulnerabilities

- If it is a minor flaw/bug, [create an issue](https://github.com/burgspoody/macnite/issues/new/choose) on GitHub.
- But do not create an issue on GitHub for security vulnerabilities that may endanger the user or MacNite itself.
  - Instead, email privately at macnite.email@icloud.com about: 
    - What the security vulnerability is about
    - Where the security vulnerability occured
    - How you found it
    - Potential solutions to fix it
  - This might prevent the people from accessing the major security flaw.

## What We Accept
- Bugs and issues with MacNite
- New apps to applist.json
- Fixed download URLs
- Improved app descriptions
- Documentation improvements
- Suggestions for MacNite or its [website](https://burgspoody.github.io/macnite)

## What We Don't Accept
- Changes to core logic of MacNite
- Changes to the [website](https://burgspoody.github.io/macnite)
- UI changes without prior discussion
- Apps that are paid only
- Apps that are not available for Apple Silicon (Intel-only apps)
